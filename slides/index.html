CONCEPTS.md — delivery flow
Shared vocabulary

PR lifecycle, precisely

Cycle Time
Creation → merge. Only defined for Merged pull requests.
Time to First Review
Creation → the earliest Review on that PR.
Time in Review
Earliest Review → merge. Distinct from Cycle Time.
Allowlisted Author
Flag controlling who counts toward commit-activity metrics — not applied to PR breakdowns.

internal/storage/sqlite.go
One file, six tables

SQLite storage

storage.Open(path)
-- created automatically, idempotent
CREATE TABLE IF NOT EXISTS repositories
CREATE TABLE IF NOT EXISTS authors
CREATE TABLE IF NOT EXISTS commits
CREATE TABLE IF NOT EXISTS file_changes
CREATE TABLE IF NOT EXISTS pull_requests
CREATE TABLE IF NOT EXISTS reviews
Default path git-statistics.db, override with -db.

No manual setup — opening the DB runs the schema and the app is ready to sync.

config.yaml
Setup, step one

Config + secrets

config.yaml
bitbucket:
  workspace: my-workspace
  repos:
    - "*"          # or named repos
sync_interval_minutes: 30
authors:
  - "*"          # or named authors
Secrets stay out of YAML — set as an env var:
$ env:BITBUCKET_API_TOKEN = "xxxx"
Required, or config.Load errors out. Sent as Authorization: Bearer <token> on every request.

10 / 14
cmd/server/main.go
What happens on boot

Startup sequence

  • Load YAML config + env secrets, validate
  • Open / create SQLite DB, run schema
  • Build Bitbucket client (token auth) + syncer
  • Resolve repos: ["*"] via the Bitbucket API if used
  • Start background scheduler (every sync_interval_minutes)
  • Serve dashboard routes via internal/web
